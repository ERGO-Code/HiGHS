    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-highs---linear-optimization-software" class="anchor" aria-hidden="true" href="#highs---linear-optimization-software"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>HiGHS - Linear optimization software</h1>
<p><a href="https://travis-ci.org/ERGO-Code/HiGHS" rel="nofollow"><img src="https://camo.githubusercontent.com/1503c485b98d046ebd757c49ce9b4554208b07eb/68747470733a2f2f7472617669732d63692e6f72672f4552474f2d436f64652f48694748532e7376673f6272616e63683d6d6173746572" alt="Build Status" data-canonical-src="https://travis-ci.org/ERGO-Code/HiGHS.svg?branch=master" style="max-width:100%;"></a></p>
<p>HiGHS is a high performance serial and parallel solver for large scale
sparse linear programming (LP) problems of the form</p>
<pre><code>Maximize c^Tx subject to L &lt;= Ax &lt;= U; l &lt;= x &lt;= u
</code></pre>
<p>It is written in C++ with OpenMP directives. It is based on the dual
revised simplex method implemented in HSOL, exploiting parallelism using either "parallel
minor iterations" (PAMI) or "single iteration parallelism" (SIP). A
full technical reference to PAMI and SIP is</p>
<p>Parallelizing the dual revised simplex method
Q. Huangfu and J. A. J. Hall
Mathematical Programming Computation, 10 (1), 119-142, 2018.
DOI: 10.1007/s12532-017-0130-5</p>
<p><a href="http://www.maths.ed.ac.uk/hall/HuHa13/" rel="nofollow">http://www.maths.ed.ac.uk/hall/HuHa13/</a></p>
<p>HSOL was originally written by Qi Huangfu, with features such as
presolve, crash and advanced basis start added by Julian Hall and Ivet
Galabova.</p>
<p>HSOL has been developed and tested on various linux installations
using both the GNU (g++) and Intel (icc) C++ compilers.</p>
<h2><a id="user-content-compilation" class="anchor" aria-hidden="true" href="#compilation"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Compilation</h2>
<p>HiGHS uses CMake as build system. To compile the run you need to setup
a build directory and define your build configuration:</p>
<pre><code>mkdir build
cd build
cmake .. [add otional parameters here]
</code></pre>
<p>Afterwards you may compile the code wrt the set configuration using your
defined build generator, e.g. <code>make</code>:</p>
<pre><code>make
</code></pre>
<h2><a id="user-content-useful-options" class="anchor" aria-hidden="true" href="#useful-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Useful options</h2>
<p>Set custom options with <code>-D&lt;option&gt;=&lt;value&gt;</code> during the configuration step (<code>cmake ..</code>):</p>
<ul>
<li><code>OLD_PARSER</code>:
on: Uses the original fixed-format MPS file reader
off: Uses the new free-format MPS file reader (which requires Boost)</li>
<li><code>OPENMP</code>:
on: Causes OpenMP to be used if available - cmake checks for this</li>
<li><code>SCIP_DEV</code>:
on: Adds some printing for SCIP interface development when, otherwise, all HiGHS output is suppressed</li>
<li><code>HiGHSDEV</code>:
on: Includes a lot of testing and development code which should not be used in "production" or when running optimized code</li>
<li><code>HiGHSRELEASE</code>:
on: Defined when CMAKE_BUILD_TYPE=Release</li>
</ul>
<h2><a id="user-content-testing" class="anchor" aria-hidden="true" href="#testing"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Testing</h2>
<p>To perform a quick test whether the compilation was successful, run</p>
<pre><code>ctest
</code></pre>
<h2><a id="user-content-run-time-options" class="anchor" aria-hidden="true" href="#run-time-options"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Run-time options</h2>
<p>In the following discussion, the name of the executable file generated
is assumed to be <code>HiGHS</code>.</p>
<p>HiGHS can only read plain text MPS files, and the following command
solves the model in <code>ml.mps</code></p>
<pre><code>highs -f ml.mps
</code></pre>
<h2><a id="user-content-usage" class="anchor" aria-hidden="true" href="#usage"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Usage</h2>
<pre><code>usage: highs [options] -f fName.mps 

Options:
    -p mode  : use presolve mode. Values:
             : Off On
    -c mode  : use crash mode to mode. Values:
             : Off LTSSF LTSSF1 LTSSF2 LTSSF3 LTSSF4 LTSSF5 LTSSF6 LTSSF7
    -e edWt  : set edge weight to edWt. Values:
             : Dan Dvx DSE DSE0 DSE1
    -s       : use option sip
    -m [cut] : use pami. Cutoff optional double value.
    -t fName : use pami with partition file fName
    -d       : debug mode on
</code></pre>
<p>Run-time options <code>-p</code> and <code>-s</code> direct HiGHS to use PAMI or SIP.</p>
<p>When compiled with the OpenMP directives invoked, the number of
threads used at run time is the value of the environment variable
<code>OMP_NUM_THREADS</code>. For example, to use HiGHS with PAMI and eight
threads to solve <code>ml.mps</code> execute</p>
<pre><code>export OMP_NUM_THREADS=8
highs -m -f ml.mps
</code></pre>
<p>If <code>OMP_NUM_THREADS</code> is not set, either because it has not been set or
due to executing the command</p>
<pre><code>unset OMP_NUM_THREADS
</code></pre>
<p>then all available threads will be used.</p>
<h2><a id="user-content-observations" class="anchor" aria-hidden="true" href="#observations"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Observations</h2>
<p>When compiled without the OpenMP directives, or if run with
<code>OMP_NUM_THREADS=1</code>, HiGHS is serial. The <code>-sip</code> run-time option will not
affect performance. The <code>-pami</code> run-time option will cause HiGHS to use
serial minor iterations and, although this could lead to better
performance on some problems, performance will typically be
diminished.</p>
<p>When compiled with the OpenMP directives and <code>OMP_NUM_THREADS&gt;1</code> or
unset, HiGHS will use multiple threads if the <code>-pami</code> or <code>-sip</code> run-time
option is specified. If <code>OMP_NUM_THREADS</code> is unset, HiGHS will try to use
all available threads so performance may be very slow. Although the
best value will be problem and architecture dependent,
<code>OMP_NUM_THREADS=8</code> is typically a good choice. Although HiGHS is slower
when run in parallel than in serial for some problems, it is typically
faster, with the <code>-pami</code> option usually faster than the <code>-sip</code> option.</p>
